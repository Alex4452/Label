Данная программа содержит в себе label.dll, которая разбирает файлы, содержащие метки.

В первую очередь необходимо отправить в библиотеку на разбор файлы, содержащие в себе доступные метки,
т.е. уровни доступа, отсеки и группы, которые будет в себе содержать метка объекта.
Доступная метка состоит из номера, длинной записи имени и короткой записи имени, так же метка группы 
содержит в себе дополнительное поле содержашее полное длинную запись родительской группы. Каждая метка
находится на отдельной строке. Составляющие метки разделяются двоеточием, пример: 
номер:длинная_запись:короткая_запись:родительская_группа(актуально только для меток групп)

Для доступных меток создаются хеш таблицы, индексы в который генерируются из длинной записи имени метки.

Затем загружается файл с метками объектов, который содержит в себе метки объектов, состоящих 
из доступных меток.
Каждая метка записывается на отдельной строке, и содержит в себе: уровень доступа, отсеки и группы.
Уровень доступа может содержать в себе либо одну метку с уровнем доступа для сеанса, либо три метки, 
где первая обозначает метку с уровнем доступа для сеанса, вторая - минимальный уровень доступа данного
объекта, третья максимальный уровень доступа объекта.
Отсеки и группы записываются через запятую.
Информация о метках берется из хеш таблиц с доступными метками. В метки объекта записывается длинная 
запись имени доступной метки. 
Пример метки объекта:
уровень_доступа_сеанса, минимальный_уровень, максимальный_уровень:отсек1, отсек2: группа
